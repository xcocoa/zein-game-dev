# StoryForge 商业化落地 & 团队组建方案

## 一、项目定位

**一句话**：让游戏社区玩家和网文读者，一键把喜欢的小说变成可玩的互动视觉小说。

**产品形态**：Web SaaS 平台（上传小说 → AI 处理 → 在线玩/下载游戏）

---

## 二、从 CLI 到 SaaS 的产品路线（1-2个月）

### Phase 0 — 当前状态（已完成）
- CLI 工具，本地 Ollama 模型
- 文本输入 → Ren'Py 游戏输出
- 纯文字，无图无音

### Phase 1 — Web 化 MVP（第 1-3 周）
**目标**：用户在浏览器里完成"上传小说 → 等待 → 在线玩"的完整流程

| 任务 | 具体内容 | 负责角色 |
|------|----------|----------|
| 后端 API | FastAPI 包装现有 pipeline，异步任务队列（Celery/ARQ） | 后端 |
| Web 播放器 | 基于 Web 的视觉小说播放器（替代 Ren'Py 桌面端），可用 monogatari.js 或自研 | 前端 |
| 基础页面 | 上传页、处理进度页、游戏播放页、我的作品列表 | 前端 |
| LLM 后端切换 | 支持云端 API（DeepSeek/Qwen API），不再依赖本地 Ollama | AI |
| 部署 | Docker 化 + 云服务器部署（阿里云/腾讯云轻量） | 后端 |

**关键决策**：Web 播放器方案
- 方案 A：用 monogatari.js（开源 Web 视觉小说引擎），生成器输出 monogatari 脚本而非 .rpy
- 方案 B：自研轻量播放器（React/Vue），对话框+选择+变量系统，更可控
- **建议选 B**：自研轻量播放器，核心逻辑不超过 500 行，完全可控，用户体验更好

### Phase 2 — 体验增强（第 3-6 周）
**目标**：从"能用"到"好用"，形成口碑传播的基础

| 任务 | 具体内容 |
|------|----------|
| AI 生图 | 接入 SD/FLUX API，自动生成角色立绘和场景背景 |
| 剧情编辑器 | 可视化编辑 AI 生成的剧情线——调整对话、修改选择、增删分支 |
| 社区功能 | 作品广场（发布/浏览/点赞）、排行榜 |
| 分享 | 一键生成分享链接，可嵌入论坛/社交媒体 |

### Phase 3 — 商业化（第 6-8 周）
**目标**：跑通付费链路

| 层级 | 价格 | 内容 |
|------|------|------|
| 免费版 | 0 | 每月 3 次转换，纯文字，带水印 |
| 基础版 | 19.9/月 | 无限转换，AI 生图，去水印 |
| 高级版 | 49.9/月 | 自定义画风、BGM、导出 Ren'Py 项目包、优先队列 |

## 三、团队组建（3-5人）

### 需要的角色和人选画像

#### 1. AI 工程师（1人）
**做什么**：优化 LLM 管线质量、接入云端模型、实现 AI 生图
**画像**：
- 玩过 LLM 应用开发（LangChain/原生 API 都行）
- 对 prompt engineering 有实践经验
- 加分项：了解 Stable Diffusion / ComfyUI

**吸引点**：
- "不是又一个 chatbot，是结构化信息抽取 + 多步推理 + 多模态生成的完整管线"
- "你写的 prompt 直接影响游戏质量，效果可见、可玩"

#### 2. 全栈/前端工程师（1人）
**做什么**：Web 播放器、上传/管理页面、剧情可视化编辑器
**画像**：
- React/Vue 熟练，有独立做过完整项目
- 对游戏/互动体验有兴趣
- 加分项：做过编辑器类产品（拖拽、节点图）

**吸引点**：
- "做一个可视化剧情编辑器，不是普通的 CRUD 后台"
- "前端播放器要处理对话动画、选择分支、变量系统，技术有深度"

#### 3. 后端工程师（1人）
**做什么**：API 服务、任务队列、用户系统、部署运维
**画像**：
- Python（FastAPI/Django）或 Go
- 有异步任务处理经验
- 加分项：有 SaaS 产品经验

**吸引点**：
- "从 0 到 1 搭一个 AI SaaS 的后端架构，不是给现有系统加功能"

#### 4. 产品/运营（0-1人，可由创始人兼）
**做什么**：用户调研、社区运营、内容策划
**画像**：
- 本身是游戏玩家或网文读者
- 有社区运营经验（论坛、Discord、QQ群）

### 去哪里找人

| 渠道 | 方式 | 优先级 |
|------|------|--------|
| 身边同事/朋友 | 直接聊，展示 Demo | 最高 |
| V2EX / 即刻 | 发帖"找人一起做 AI 互动小说项目" | 高 |
| GitHub | 开源核心代码，吸引 contributor | 高 |
| 独立开发者社群 | 各种微信群、Discord | 中 |
| 高校 AI 社团 | 找有热情的学生参与 | 中 |

### 招人话术模板

> **找 AI 工程师**：
> "我在做一个 AI 项目——把小说自动转成可玩的视觉小说游戏。不是套 API 做 chatbot，是 6 阶段 LLM 编排管线 + 编译器级代码生成。MVP 已经跑通了，导入《罗生门》3 分钟出可玩游戏。现在要做云端化和 AI 生图，需要一个对 LLM 应用有实战经验的人。有兴趣聊聊？"

> **找前端**：
> "我在做一个项目，需要做 Web 视觉小说播放器 + 可视化剧情编辑器。不是普通 CRUD，是游戏级的前端交互——对话动画、分支选择、节点图编辑。技术栈自由，核心是把体验做好。项目已经有后端管线了，现在缺前端把它变成产品。"

## 四、技术架构演进

```
当前（CLI）:
  小说.txt → [本地 Ollama] → StoryForge Pipeline → .rpy 文件 → Ren'Py 桌面运行

目标（SaaS）:
  用户上传 → [Web API] → [任务队列] → [云端 LLM] → StoryForge Pipeline
                                                         ↓
                                              → 故事 JSON（AST）
                                                    ↓
                                           → Web 播放器在线玩
                                           → 可视化编辑器修改
                                           → 导出 Ren'Py 项目包
                                           → AI 生图（角色/背景）
```

### 需要新增的技术模块

| 模块 | 技术选型 | 工作量 |
|------|----------|--------|
| Web API | FastAPI + Pydantic（复用现有 models） | 小 |
| 任务队列 | ARQ（轻量 Redis 队列）或 Celery | 小 |
| Web 播放器 | React + TypeScript，自研 | 中 |
| 剧情编辑器 | React Flow（节点图）+ 表单编辑 | 中 |
| 云端 LLM | DeepSeek API / 通义千问 API | 小 |
| AI 生图 | 接 LiblibAI / Replicate API | 小 |
| 用户系统 | 微信登录 + JWT | 小 |
| 存储 | PostgreSQL + 对象存储（OSS） | 小 |
| 部署 | Docker Compose + Nginx | 小 |

### 现有代码的复用

现有 StoryForge 代码（storyforge/ 目录）几乎 100% 可复用：
- `parser/` — LLM 解析管线，只需把 Ollama client 换成支持云端 API
- `ast/` — AST 节点定义，完全不变
- `generator/` — 代码生成器，新增 Web JSON 格式输出（与 .rpy 并行）
- `parser/models.py` — Pydantic 模型，直接作为 API 的数据契约

## 五、第一个月周计划

## 六、验证商业化的关键指标

在投入更多资源之前，用以下指标验证方向：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 注册用户 | 首月 500+ | 发布后自然增长 |
| 完成转换 | 首月 200+ 次 | 说明用户真的在用 |
| 分享率 | >10% | 用户愿意把作品分享出去 |
| 次日留存 | >20% | 不是看一眼就走 |
| 付费意愿 | 调研 >30% 表示愿付费 | 问卷 or 付费墙测试 |

如果首月能达到上述指标的一半，就值得继续投入。

## 七、风险和应对

| 风险 | 应对 |
|------|------|
| LLM 生成质量不稳定 | 提供"人工编辑"兜底，AI 生成 + 人工微调的混合模式 |
| 版权问题（用别人的小说） | 1) 鼓励原创 2) 明确免责条款 3) 支持作者授权 |
| 用户不知道怎么用 | 提供模板库（经典公版小说、示例剧情），降低冷启动门槛 |
| 云端 LLM 成本高 | 免费版限次数，用小模型压成本，付费版用大模型 |
| 团队成员精力不足 | MVP 聚焦核心链路，砍掉非必要功能，3人也能跑通 |

## 八、给领导/投资人的一页纸

### StoryForge — AI 互动叙事引擎

**市场机会**：中国网络文学用户 5 亿+，视觉小说/Galgame 玩家群体持续增长，两个群体高度重叠但缺乏连接工具。

**产品**：Web 平台，用户上传小说文本，AI 自动生成可玩的互动视觉小说（含角色、对话、选择分支、多结局），支持在线游玩和分享。

**技术壁垒**：
- 6 阶段 LLM 编排管线（非简单 API 调用）
- 编译器级 AST 中间表示（可扩展多引擎输出）
- 已验证可行性：3000 字小说 → 3 分钟出可玩游戏

**进展**：CLI MVP 已完成，核心管线跑通，104 个测试覆盖。

**计划**：1-2 个月内上线 Web 版，面向游戏社区和网文社区获取种子用户。

**商业模式**：SaaS 订阅（免费版引流 + 付费版变现），后续扩展 AI 生图、BGM、导出等增值服务。

**需要的支持**：3-5 人小团队（AI + 前端 + 后端），少量云服务器资源。
